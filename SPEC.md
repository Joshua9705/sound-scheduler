# 音控團隊自動排班系統 - 需求規格

## 概述
為教會音控團隊建立一套獨立的自動排班 Web 應用程式，使用 Next.js + SQLite。

## 排班規則

### 基本規則
1. **排班頻率**：每季排一次（一次產出 3 個月排班表）
2. **每月上限**：
   - 4 週月 → 每人最多排 **2 次**
   - 5 週月 → 每人最多排 **3 次**
3. **時段限制**：每人只能排到其「可排時段」的場次

### 等級搭配規則
- 等級 1-4（1 最高、4 最低）
- **等級 3 或 4 的人，同一場次必須有等級 1 或 2 的人一起**
- 🔰「學習中」的角色能力：可以排該角色，但同樣需要等級 1-2 搭配

### 週四晚上特殊規則
- 所有人都可以報名（不受平時時段限制）
- 真的沒人可用時才空缺

## 功能需求

### 1. 成員管理頁面
- 新增/編輯/刪除成員
- 設定：姓名、等級(1-4)、可排時段（多選）、可擔任角色（PA/Stage/線上，含「學習中」標記）
- 預載上述 13 位成員資料

### 2. 排班產生頁面
- 選擇起始月份（預設下一季）
- 一鍵產生 3 個月排班表
- 自動排班演算法需滿足所有規則
- 線上角色可設定該場是否需要（開關）

### 3. 排班表檢視頁面
- 月份切換
- 以週為單位，顯示每個時段的人員配置
- 清楚標示角色（PA / Stage / 線上）
- 可手動調整（拖拽或下拉選單換人）
- 違反規則時顯示警告

### 4. 儀表板
- 各成員本季排班次數統計
- 公平性指標（排班次數分布）

## 技術規格
- **框架**：Next.js 15 (App Router)
- **資料庫**：Turso（libSQL / SQLite 相容）— `@libsql/client`
- **樣式**：Tailwind CSS
- **部署**：Vercel（Serverless）
- **語言**：繁體中文介面
- **專案目錄**：`/home/alanclaw/.openclaw/workspace/sound-scheduler`

## 環境變數
```
TURSO_DATABASE_URL=libsql://<REDACTED_DB_URL>
TURSO_AUTH_TOKEN=<REDACTED_TOKEN>
```

## 資料庫 Schema（已在 Turso 建立完成）

Tables: members, time_slots, member_slots, roles, member_roles, schedules, assignments, slot_role_requirements, configurations

所有 DB 操作必須使用 `@libsql/client` 的非同步 API（`await db.execute()`、`await db.batch()`）。

## UI 風格
- 深色主題（Dark mode）
- 乾淨、現代感
- 響應式設計（手機也能看）
